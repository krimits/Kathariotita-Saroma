# Dashboard Αποκομιδής Ογκωδών - Σχεδιασμός

## Στόχος

Το dashboard πρέπει να δίνει στα ανώτερα στελέχη του Δήμου Θεσσαλονίκης καθαρή εικόνα για την αποκομιδή ογκωδών αντικειμένων:

- Πόσα σημεία εκκρεμούν.
- Πόσα σημεία συλλέχθηκαν.
- Από ποια πηγή προέρχεται ο φόρτος.
- Ποιες οδοί συγκεντρώνουν τις περισσότερες εκκρεμότητες.
- Με τι υπόλοιπο περνά η υπηρεσία στην επόμενη εβδομάδα.
- Αν η επιχειρησιακή κατάσταση είναι άριστη, καλή ή κόκκινη.

Το dashboard δεν πρέπει να εμφανίζει απλώς το Excel με πιο όμορφο τρόπο. Πρέπει να μειώνει τον θόρυβο και να μετατρέπει τις καθημερινές καταχωρήσεις σε επιχειρησιακά συμπεράσματα.

## Επιχειρησιακοί Ορισμοί

### Κανόνες χρωμάτων στο φύλλο (canonical / ευθυγράμμιση με extractor)

Οι κανόνες που εφαρμόζει το σύστημα ανάγνωσης (Google Sheets API + local Excel) είναι οι εξής:

| Εμφάνιση στο φύλλο | Τεχνική ερμηνεία |
|---|---|
| Πορτοκαλί / κίτρινο φόντο ημέρας (F:L) + **μαύρο / σκούρο κείμενο**, χωρίς ταύτιση | Αριθμητική καταχώριση χωρίς ταύτιση στην ίδια γραμμή (C:L)· καταχώρηση **επόπτη**. Πηγή `supervisor`. |
| Πορτοκαλί / κίτρινο φόντο ημέρας + **κόκκινο κείμενο**, χωρίς ταύτιση | Ίδιο κριτήριο «χωρίς ταύτιση»· καταχώρηση από χειριστή μέσω συστήματος «Βελτιώνω την πόλη μου». Πηγή `veltio`. |
| Κόκκινο **φόντο** ημέρας (χωρίς ταύτιση) | Εξακολουθεί να ταξινομείται ως `veltio` για συμβατότητα με παλιότερα φύλλα. |
| Πορτοκαλί / κίτρινο χωρίς ταύτιση (γενικά) | Η υποκατάσταση επόπτη vs ΒΠΠΜ γίνεται από το **χρώμα γραμματοσειράς** όταν το φόντο είναι στην οικογένεια πορτοκαλί/κίτρινο· αν λείπει έγχρωμο φόντο στο αποθηκεμένο `.xlsx` (conditional formatting), το τοπικό extractor χρησιμοποιεί κυρίως το κείμενο + theme Excel (π.χ. theme 1 = σκούρο κείμενο → επόπτης). |
| Πράσινο στο φύλλο | Στη λογική του dashboard **δεν σημαίνει μόνο του «ολοκληρώθηκε»**. Τεχνικά το σύστημα θεωεί **«συλλεχθέν» / ταύτιση** όταν ο **ίδιος αριθμός** εμφανίζεται **πάνω από μία φορά** στην ίδια γραμμή στις στήλες C:L (αντιστοίχιση εκκρεμότητας με ημέρα ή μεταξύ δύο ημερών). Μοναδικός αριθμός σε πράσινο φόντο χωρίς διπλότυπο δεν ταξιμοποιείται ως ολοκληρωμένη ταύτιση. |
| Μουσταρδί / στήλες προηγούμενων εβδομάδων (C:E) | Αριθμός σε εκκρεμότητα προηγούμενων εβδομάδων **χωρίς** ταύτιση με άλλη στήλη στην ίδια γραμμή· στο μοντέλο δεδομένων η πηγή είναι `previous_weeks`. |

Στο UI το label **«Συλλέχθηκε»** αντιστοιχεί σε εγγραφές με **εντοπισμένη ταύτιση αριθμού στην γραμμή** (όπως στο φύλλο)· δεν υποκαθιστά χειροκίνητη επιβεβαίωση διαδρομής οχήματος.

### Κατάσταση σημείου (ενιαίο μοντέλο)

| Κατάσταση (`status`) | Κριτήριο | Ερμηνεία |
|---|---|---|
| Εκκρεμεί (`pending`) | Αριθμός σε ημέρα χωρίς διπλότυπο στη γραμμή, ή σε C:E χωρίς διπλότυπο | Ανοιχτό σημείο στο πλαίσιο της εβδομάδας |
| Συλλεχθέν / ταυτισμένο (`collected`) | Ο ίδιος `point_id` εμφανίζεται ≥2 φορές στη γραμμή (C:L) | Αντιστοίχιση όπως στο conditional formatting του φύλλου |

### Πηγή πληροφορίας (`source`)

| Τιμή | Συνήθης συσχέτιση στο φύλλο | Ερμηνεία |
|---|---|---|
| `veltio` | Πορτοκαλί/κίτρινο φόντο ημέρας με **κόκκινο κείμενο**, ή κόκκινο φόντο ημέρας | Αίτημα / καταχώρηση «Βελτιώνω την πόλη μου» |
| `supervisor` | Πορτοκαλί/κίτρινο φόντο ημέρας με **μαύρο / σκούρο κείμενο** (ή theme σκούρου κειμένου στο `.xlsx`) | Επιχειρησιακή διαπίστωση επόπτη |
| `previous_weeks` | Στήλες προηγούμενων εβδομάδων C:E χωρίς ταύτιση | Μουσταρδί / μεταφορά από προηγούμενες εβδομάδες |
| `unknown` | Συνδυασμός χρωμάτων εκτός κανόνων ή ασαφές κείμενο | Απαιτείται επαλήθευση στο φύλλο |

### Τρόποι δεδομένων dashboard: hybrid vs μόνο Google

| Τρόπος | Περιγραφή |
|---|---|
| **Hybrid** (προεπιλογή όταν υπάρχουν και τα δύο) | Τρέχουσα κατάσταση εκκρεμοτήτων από Google Sheets + συλλεχθέντα που προκύπτουν από το Google όπως παραπάνω· επιπλέον **ιστορικό συλλεχθέντων** από τοποπικό `.xlsx` για κλειδιά που δεν είναι πλέον εκκρεμή στο Google (αποφυγή διπλομέτρησης). Τα σύνολα `collected` / `total_records` **δεν είναι υποχρεωτικά ίδια** με pure-Google μόνο για την τρέχουσα εβδομάδα. |
| **Μόνο Google** | Σύγκριση ενός προς ένα με το alive Sheet: χρησιμοποίησε extractor Google + `verify_live_dashboard.py --mode google-only` (χωρίς στόχους hybrid για τα συγκεντρωτικά). |

Για έλεγχο καθυστέρησης ενημέρωσης: το API κάνει περιοδική ανανέωση από Google· μέγιστη ορατή καθυστέρηση ≈ διάστημα refresh backend + διάστημα poll frontend (βλ. `DASHBOARD_REFRESH_SECONDS`, `POST /api/refresh`).

### Αξιολόγηση εβδομάδας

Η εβδομάδα αξιολογείται από το υπόλοιπο εκκρεμοτήτων που μεταφέρεται στην επόμενη εβδομάδα.

| Υπόλοιπο εκκρεμοτήτων | Κατάσταση | Χρώμα dashboard |
|---:|---|---|
| 0-100 | Άριστη κατάσταση | Πράσινο |
| 101-299 | Καλή / διαχειρίσιμη κατάσταση | Πορτοκαλί |
| 300+ | Κόκκινη εβδομάδα / επιχειρησιακή πίεση | Κόκκινο |

## Προτεινόμενο Μοντέλο Δεδομένων

Κάθε γραμμή του Excel πρέπει να μετατρέπεται σε κανονικοποιημένη εγγραφή:

| Πεδίο | Περιγραφή |
|---|---|
| `week_start` | Ημερομηνία έναρξης εβδομάδας |
| `week_end` | Ημερομηνία λήξης εβδομάδας |
| `street` | Οδός |
| `street_normalized` | Καθαρισμένη ονομασία οδού για ομαδοποίηση |
| `day` | Ημέρα καταχώρησης ή συλλογής |
| `point_id` | Αριθμητικό σημείο/κωδικός από το κελί |
| `status` | `pending` ή `collected` |
| `source` | `veltio`, `supervisor`, `previous_weeks` ή `unknown` |
| `age_bucket` | Τρέχουσα εβδομάδα, προηγούμενη 1η, 2η, 3η |
| `carried_to_next_week` | Αν το σημείο μένει υπόλοιπο για την επόμενη εβδομάδα |

Αυτό το μοντέλο επιτρέπει καθαρά φίλτρα, συγκρίσεις στον χρόνο και αξιόπιστη μέτρηση απόδοσης.

### Executive KPIs

| KPI | Τι δείχνει | Γιατί έχει αξία |

| Υπόλοιπο επόμενης εβδομάδας | Πόσα μεταφέρονται | Κύριος δείκτης πίεσης |
| Ποσοστό ολοκλήρωσης | Συλλεχθέντα / συνολικά σημεία | Δείχνει αποτελεσματικότητα |
| Νέες εκκρεμότητες | Πόσα νέα σημεία μπήκαν | Δείχνει εισερχόμενο φόρτο |
| Εκκρεμότητες από εφαρμογή | Κόκκινα σημεία | Δείχνει πίεση από πολίτες |
| Εκκρεμότητες από επόπτες | Πορτοκαλί σημεία | Δείχνει εσωτερική επιχειρησιακή εικόνα |

|---|---|---|

| Ρυθμός συλλογής | Συλλεχθέντα ανά εργάσιμη ημέρα | Δείχνει επιχειρησιακή δυνατότητα |

| Επαναλαμβανόμενες οδοί | Οδοί με υψηλή εμφάνιση σε πολλές εβδομάδες | Δείχνει μόνιμα προβληματικά σημεία |

## Καρτέλες Dashboard

### 1. Επισκόπηση Διοίκησης

Πρώτη οθόνη για Δήμαρχο, Αντιδήμαρχο, Γενική Διεύθυνση ή ανώτερα στελέχη.

Περιεχόμενο:

- KPI cards:
  - Σύνολο εκκρεμών.
  - Συλλεχθέντα εβδομάδας.
  - Υπόλοιπο για επόμενη εβδομάδα.
  - Κατάσταση εβδομάδας: Άριστη / Καλή / Κόκκινη.
- Donut chart: εκκρεμότητες ανά πηγή.
- Bar chart: συλλεχθέντα ανά ημέρα.
- Top 10 οδοί με περισσότερες εκκρεμότητες.
- Σύντομο συμπέρασμα σε φυσική γλώσσα:
  - "Η εβδομάδα κλείνει σε καλή κατάσταση, με 184 εκκρεμότητες για μεταφορά."
  - "Η εβδομάδα είναι κόκκινη, καθώς το υπόλοιπο ξεπερνά το όριο των 300 σημείων."

### 2. Ημερήσια Ροή

Για παρακολούθηση μέσα στην εβδομάδα.

Περιεχόμενο:

- Γραμμικό διάγραμμα εκκρεμοτήτων ανά ημέρα.
- Bar chart:
  - Νέα σημεία ανά ημέρα.
  - Συλλεχθέντα ανά ημέρα.
- Φίλτρο ημέρας.
- Πίνακας με τις οδούς που παραμένουν ανοιχτές ανά ημέρα.

### 3. Οδοί Υψηλής Πίεσης

Για επιχειρησιακή στόχευση.

Περιεχόμενο:

- Top οδοί με περισσότερες εκκρεμότητες.
- Top οδοί με περισσότερες επαναλήψεις.
- Διαχωρισμός ανά πηγή:
  - Βελτιώνω την πόλη μου.
  - Επόπτες Καθαριότητας.
- Δυνατότητα drill-down σε συγκεκριμένη οδό.

### 4. Παλαιότητα Εκκρεμοτήτων

Για να φαίνεται αν παλιά σημεία μένουν πίσω.

Περιεχόμενο:

- Εκκρεμότητες προηγούμενων εβδομάδων 1η, 2η, 3η.
- Πόσες παλιές εκκρεμότητες συλλέχθηκαν.
- Πόσες παλιές εκκρεμότητες μεταφέρονται.
- Alarm για εκκρεμότητες 2ης ή 3ης προηγούμενης εβδομάδας.

### 5. Ιστορική Τάση

Θα ενεργοποιηθεί όταν υπάρχουν αρκετές εβδομάδες δεδομένων.

Περιεχόμενο:

- Υπόλοιπο ανά εβδομάδα.
- Ποσοστό ολοκλήρωσης ανά εβδομάδα.
- Νέα σημεία ανά εβδομάδα.
- Συλλεχθέντα ανά εβδομάδα.
- Κυλιόμενος μέσος όρος 4 εβδομάδων.
- Αυτόματη ένδειξη:
  - Βελτιώνεται.
  - Χειροτερεύει.
  - Παραμένει σταθερό.

## Φίλτρα

Τα βασικά φίλτρα πρέπει να είναι λίγα και επιχειρησιακά χρήσιμα:

- Εβδομάδα.
- Ημέρα.
- Οδός.
- Κατάσταση: εκκρεμεί / συλλέχθηκε.
- Πηγή: Βελτιώνω την πόλη μου / Επόπτες / προηγούμενες εβδομάδες (`previous_weeks`).
- Παλαιότητα: τρέχουσα εβδομάδα / προηγούμενη 1η / 2η / 3η.
- Κατάσταση εβδομάδας: άριστη / καλή / κόκκινη.

## Κανόνες Συμπερασμάτων

### Κατάσταση εβδομάδας

```text
αν υπόλοιπο <= 100:
    Άριστη κατάσταση
αν υπόλοιπο >= 101 και υπόλοιπο < 300:
    Καλή / διαχειρίσιμη κατάσταση
αν υπόλοιπο >= 300:
    Κόκκινη εβδομάδα
```

### Βελτίωση ή επιδείνωση

Για να υπολογιστεί αξιόπιστα χρειάζεται ιστορικό. Προτεινόμενος κανόνας:

```text
αν υπόλοιπο τρέχουσας εβδομάδας < υπόλοιπο προηγούμενης εβδομάδας:
    Βελτίωση
αν υπόλοιπο τρέχουσας εβδομάδας > υπόλοιπο προηγούμενης εβδομάδας:
    Επιδείνωση
αν η διαφορά είναι μικρή:
    Σταθερή κατάσταση
```

Για να μη βγαίνουν λανθασμένα συμπεράσματα από μία τυχαία δύσκολη εβδομάδα, προτείνεται να χρησιμοποιηθεί και κυλιόμενος μέσος όρος 4 εβδομάδων.

## Τεχνικές Παρατηρήσεις Από Το Τρέχον Excel

- Το αρχείο περιέχει ένα φύλλο με τίτλο `ΣΥΛΛΟΓΗ ΟΓΚΩΔΗ`.
- Υπάρχουν 503 μη κενές γραμμές οδών και 369 μοναδικές οδοί.
- Οι οδοί περιέχουν παραλλαγές γραφής, τόνους, κενά και μικρές αποκλίσεις. Χρειάζεται καθαρισμός ονομάτων για σωστή ομαδοποίηση.
- Τα χρώματα φαίνεται να παράγονται εν μέρει από conditional formatting. Για dashboard παραγωγής πρέπει να διαβάζεται αξιόπιστα η τελική κατάσταση κάθε κελιού.
- Αν το dashboard συνδεθεί απευθείας με Google Sheets, η καλύτερη λύση είναι να διαβάζονται μαζί οι τιμές και τα effective cell formats από το Google Sheets API.

## Προτεινόμενη Υλοποίηση Σε Φάσεις

### Φάση 1 - Σταθεροποίηση Δεδομένων

Παραδοτέα:

- Ανάγνωση του Excel ή Google Sheet.
- Μετατροπή σε καθαρό πίνακα εγγραφών.
- Αναγνώριση κατάστασης: εκκρεμεί / συλλέχθηκε.
- Αναγνώριση πηγής: εφαρμογή / επόπτης.
- Καθαρισμός οδών για ομαδοποίηση.

### Φάση 2 - Πρώτο Dashboard

Παραδοτέα:

- Καρτέλα Επισκόπησης Διοίκησης.
- Καρτέλα Ημερήσιας Ροής.
- Καρτέλα Οδών Υψηλής Πίεσης.
- Βασικά φίλτρα.
- Εξαγωγή συμπεράσματος εβδομάδας.

### Φάση 3 - Ιστορικότητα

Παραδοτέα:

- Αποθήκευση εβδομαδιαίων snapshots.
- Σύγκριση εβδομάδα με εβδομάδα.
- Ιστορικά trends.
- Αυτόματη ένδειξη βελτίωσης/επιδείνωσης.

### Φάση 4 - Επιχειρησιακές Προβλέψεις

Παραδοτέα:

- Εκτίμηση αναμενόμενου υπολοίπου.
- Εντοπισμός οδών με επαναλαμβανόμενο πρόβλημα.
- Πρόταση προτεραιοποίησης διαδρομών.
- Προειδοποίηση όταν η εβδομάδα κινείται προς κόκκινη κατάσταση.

## Πρώτη Έκδοση Που Αξίζει Να Υλοποιηθεί

Η πρώτη πρακτική έκδοση πρέπει να είναι μικρή αλλά χρήσιμη:

1. Ανάγνωση του τρέχοντος αρχείου.
2. Υπολογισμός:
   - εκκρεμών,
   - συλλεχθέντων,
   - υπολοίπου επόμενης εβδομάδας,
   - πηγής εκκρεμοτήτων,
   - top οδών.
3. Μία οθόνη Executive Overview.
4. Μία καρτέλα με αναλυτικό πίνακα φιλτραρισμένων εγγραφών.
5. Εβδομαδιαίο snapshot για να αρχίσει να χτίζεται ιστορικό.
